1. Creating an Account

Get started with TradeLab by creating your personal account:

  1. Click the "Sign Up" button on the homepage
  2. Enter your email address and create a secure password
  3. Verify your email address through the confirmation link
  4. Alternatively you can sign up using your Google account.

2. Manual Trade Entry

Entering a Trade

Record your trades easily using our intuitive trade entry form:

Use the form fields to enter your trade details, and click "Add Trade" to add it to your account.

The trade is added to your account and displays in the recent trades table at the bottom of the page.

Notes:

  • Ensure all fields are filled in correctly before submitting the trade.
  • Use the "Edit Trade" button to make changes to a trade after it has been added.
  • Click the "Delete Trade" button to remove a trade from your account.

Tips

  • Clicking on a ticker symbol will bring up a new window with detailed information about your trade including a chart and a technical analysis meters from TradingView.

3. Trade History

The Trade History page provides a comprehensive view of all your trading activity with powerful filtering and analysis tools.

Trade Statistics Dashboard

Customize your analysis with advanced filters and sort options:

4. Analytics

The Analytics page provides powerful insights into your trading performance through comprehensive data visualization and statistical analysis.

This page provides more details at the top of the page as an overview of your trading performance.

  • Cumulative P/L: This metric shows your total profit/loss across all trades in the specified time frame (default to YTD for the current year)
  • Win Rate: This metric shows the percentage of profitable trades in the specified time frame
  • Average P/L per Trade: This metric shows the average profit/loss per trade in the specified time frame
  • Profit Factor: This metric shows the ratio of average profit per trade to the average loss per trade. A higher profit factor indicates better trading performance.
  • Total Trades: This metric shows the total number of trades in the specified time frame
  • Best Trade: This metric shows the best performing trade in the specified time frame
  • Worst Trade: This metric shows the worst performing trade in the specified time frame

Charts

This page provides a whole range of useful charts for you to analyze your performance in your specified time periods.

There are several charts available to help you understand your performance better:

  • Cumulative P/L: This charts shows the cumulative win or loss for the specified time period. The chart has an x-axis where each point is a day. This means that each point shows the cumulative P/L accounting for all trades for that corresponding day.
  • Drawdown: This charts shows the drawdown for the specified time period. The drawdown shows the highest point the cumulative P/L has dropped to over the specified time period shown as a percentage.

In the detailed statistics section, you can see your cumulative P/L, win rate, average P/L per trade, win streak, loss streak, and more.

Use these statistics to help you understand your trading performance and make informed decisions.

Analytics Best Practices

  • Regular Review: Set aside time weekly to analyze your performance metrics
  • Pattern Recognition: Use multiple timeframes to identify consistent patterns
  • Risk Management: Monitor drawdown and win rate to optimize position sizing
  • Strategy Refinement: Use trade type analysis to focus on most profitable approaches

Note: All analytics are calculated in real-time based on your trading data. The more trades you record, the more meaningful your analytics will become. Use these insights to make data-driven decisions about your trading strategy.

About TradeLab's Storage

Project Background

TradeLab started as a class project but evolved into something much more meaningful. As a junior software engineer who also trades actively, I noticed a significant gap in the market: while there are excellent trading journal solutions available, many come with substantial subscription costs that can be challenging for beginners.

Having just completed my coding classes, I decided to transform my class project into a real-world application that could help traders like myself. The goal was clear: create a comprehensive, powerful trading journal that remains accessible to everyone, especially those just starting their trading journey.

Project Vision: As both a developer and trader, I understand the importance of having robust tools without breaking the bank. TradeLab aims to provide professional-grade features while keeping costs minimal through smart technical choices.

Understanding Browser Storage

Modern web browsers offer several ways to store data locally. Let's explore each option and understand why we chose IndexedDB for TradeLab:

Local Storage

  • What it is: Simple key-value storage mechanism in the browser
  • Storage Limit: Usually 5-10 MB per domain
  • Data Persistence: Persists until explicitly cleared, survives browser restarts
  • Security: Vulnerable to XSS attacks, data stored as plain text
  • Why Not for TradeLab:
    • Limited storage capacity for trade data
    • No structured query support
    • Can't store complex data types (only strings)
    • No built-in indexing for performance

Session Storage

  • What it is: Similar to Local Storage but with session-based persistence
  • Storage Limit: Also typically 5-10 MB
  • Data Persistence: Only lasts for one browser session, cleared when tab closes
  • Security: Slightly more secure than Local Storage due to session isolation
  • Why Not for TradeLab:
    • Temporary nature unsuitable for trade records
    • Same limitations as Local Storage
    • Data loss risk when closing browser

Cache API

  • What it is: Modern API for storing network requests and responses
  • Storage Limit: Based on available disk space
  • Data Persistence: Managed by service workers, survives browser restarts
  • Security: Protected by same-origin policy, requires HTTPS
  • Why Not for TradeLab:
    • Designed for caching network resources
    • Not optimized for structured data storage
    • Lacks querying and indexing capabilities

IndexedDB (Our Choice)

  • What it is: Low-level API for client-side storage of structured data
  • Storage Limit: Generally >50MB, can extend with user permission
  • Data Persistence: Permanent until explicitly deleted, survives browser restarts
  • Security Features:
    • Protected by same-origin policy
    • Built-in transaction support
    • Data integrity protection
    • Resistant to XSS attacks
  • Perfect for TradeLab Because:
    • Supports complex data structures needed for trades
    • Provides robust querying capabilities
    • Offers excellent performance with indexes
    • Handles large datasets efficiently
    • Built-in data integrity protection

A Note on Security

While browser storage options have different security characteristics, it's important to understand that:

  • All browser storage is isolated per domain (same-origin policy)
  • No browser storage is suitable for sensitive data like passwords
  • Data should be encrypted if storing sensitive information
  • Regular backups are essential regardless of storage method

Data Protection: TradeLab implements additional security measures beyond the basic browser storage security:

  • Data validation before storage
  • Regular integrity checks
  • Automated backup reminders
  • Export functionality for manual backups

For more detailed information about browser storage, check out these resources:

  • Mozilla's IndexedDB Documentation
  • Web Storage Overview

    • Why IndexedDB?

    We chose IndexedDB as our primary storage solution for several compelling reasons:

    • Database-Like Capabilities:
      • Complex data structures support
      • Efficient indexing and querying
      • Transaction support for data integrity
      • Significant storage capacity (generally >50MB)
    • Cost Benefits:
      • No server costs passed to users
      • No need for expensive database hosting
      • Completely free for users
    • Privacy:
      • Data stays on your device
      • No external data transmission
      • Complete control over your information

    Storage Capacity Analysis

    Let's break down the storage requirements for trade data:

    Single Trade Object (Approximate Size):

    • Date: 8 bytes
    • Symbol: ~10 bytes
    • Entry/Exit Prices: 16 bytes
    • Position Size: 8 bytes
    • Notes (average): 200 bytes
    • Additional Metadata: 50 bytes
    • Total per trade: ~300 bytes

    Storage Calculations:

    • Active trader (20 trades/day): 6,000 bytes/day
    • Monthly usage: ~180KB
    • Yearly usage: ~2.2MB

    Supabase Free Tier (1GB):

    • Can support: ~500 active traders
    • Or: ~2,000 moderate traders
    • Or: ~5,000 casual traders
